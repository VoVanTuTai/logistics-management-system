Cách chạy đúng bây giờ vẫn là dev mode:

powershell -ExecutionPolicy Bypass -File .\run-all.ps1 -MobileMode lan
Hoặc nếu infra đã chạy rồi và chỉ muốn start lại service/UI:

powershell -ExecutionPolicy Bypass -File .\run-all.ps1 -MobileMode lan -SkipInfra
Script này không build Docker image cho 11 backend services. Nó chỉ:

chạy Docker cho infra dev: PostgreSQL + RabbitMQ
chạy migration / db push
start backend bằng Node/TypeScript dev runtime
start web/mobile dev servers
Phần container hiện tại chỉ dùng cho infra dependency, không phải build từng service.

Tóm lại:

Postgres/RabbitMQ  -> chạy bằng Docker Compose
Backend services   -> chạy dev bằng node/ts-node
Frontend apps      -> chạy dev server
Không build image service
Nếu chỉ cần start backend services, dùng:

powershell -ExecutionPolicy Bypass -File .\scripts\start-services-retry.ps1
Nếu chỉ cần infra:

powershell -ExecutionPolicy Bypass -File .\scripts\dev-up.ps1
Lưu ý: mình vừa sửa start-services-retry.ps1 để nó start service kiểu dev, chạy Prisma prepare trước rồi mới chạy ts-node. Không có bước Docker build service trong đó.
